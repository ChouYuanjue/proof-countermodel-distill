\section{Appendix}

\subsection{Prompt Formats}

All variants use the same fact and rule context and differ only in the target format. The prompt prefix is:
\begin{verbatim}
You are a careful logical reasoner.
Use only the numbered facts and rules below.
\end{verbatim}

The main task formats are:
\begin{itemize}
\item \textbf{answer-only}: \texttt{LABEL: <True|False|Unknown>}
\item \textbf{proof-only}: \texttt{MODE}, \texttt{LABEL}, \texttt{CHAIN}, and \texttt{WITNESS}, with \texttt{CHAIN} allowed to be \texttt{NONE}
\item \model{} and its ablations: the same four fields, with \texttt{CHAIN} used for either a derivation path or a \texttt{FAIL[...]} abstention tag
\end{itemize}

This shared prompt structure matters for the comparison. The main difference between variants is the supervision target, not the surrounding instruction.

\subsection{Training Hyperparameters}

All runs use QLoRA with:
\begin{itemize}
\item 4-bit NF4 loading,
\item LoRA rank 16,
\item LoRA alpha 32,
\item dropout 0.05,
\item max sequence length 512,
\item learning rate \(2\times 10^{-4}\),
\item one training epoch.
\end{itemize}

For 7B runs, the per-device train batch size is 2 with gradient accumulation 8.
The seeded 4k suite changes the sampled training questions across seeds while
holding the evaluation subset fixed. Experiments were run on NVIDIA A100 80GB
GPUs.

\subsection{Verifier Details}

The verifier replays generated outputs against the benchmark's symbolic structure:
\begin{itemize}
\item For \textbf{PROVE} outputs, the predicted \texttt{CHAIN} must derive the queried literal.
\item For \textbf{REFUTE} outputs, the predicted \texttt{CHAIN} must derive the opposite literal.
\item For \textbf{ABSTAIN} outputs, the \texttt{WITNESS} must describe missing support from the canonical blocked derivation, or the chain must match the canonical \texttt{FAIL[no\_rule]} target for that query.
\end{itemize}

This verifier is deliberately lightweight. It does not search for an alternative proof on the model's behalf. It only checks the consistency of the model's own evidence claim. The target constructor, however, does run a closure computation before any \texttt{FAIL[no\_rule]} target is created. Thus a bare \texttt{FAIL[no\_rule]} label is not accepted for arbitrary unknown outputs; it receives credit only when the query's canonical blocked-derivation target is also \texttt{FAIL[no\_rule]}.

\paragraph{Abstention target construction.}
The target constructor uses the following logic, with a maximum recursive
failure depth of 3 in our experiments:
\begin{verbatim}
closure = forward_chain(facts, rules)
if query in closure: emit PROVE target
if opposite(query) in closure: emit REFUTE target

def explain(target, depth):
    if target in closure:
        return PROVED
    candidates = []
    for rule in rules:
        for binding in unify(rule.consequent, target):
            missing = []
            for antecedent in rule.antecedents:
                grounded = instantiate(antecedent, binding)
                if grounded not in closure:
                    missing.append(explain(grounded, depth - 1)
                                   if depth > 0 else MISSING(grounded))
            if missing:
                candidates.append(FAIL_RULE(rule.id, missing))
    if candidates is empty:
        return FAIL_NO_RULE(target)
    return argmin(candidates,
                  key=(number_of_missing_leaves,
                       failure_depth,
                       rule_id))
\end{verbatim}

This procedure makes the path-relevance assumption explicit: a
\texttt{FAIL[ruleX]} witness must correspond to a rule whose consequent can
derive the current target under some grounding, and its missing leaves are the
antecedents that block that grounded rule application. It does not enumerate all
minimal hitting sets of all possible derivations.

\paragraph{Abstention verification.}
Given a parsed model output for a gold-\texttt{Unknown} query, the main verifier
uses:
\begin{verbatim}
canonical = explain(query, depth=3)
if mode != ABSTAIN:
    return false
if normalized(chain) == canonical.tag:
    return true
if normalized(witness) == canonical.witness:
    return true
if every canonical missing leaf is mentioned in witness:
    return true
return false
\end{verbatim}
The strict audit in Table~\ref{tab:abstention-audit} replaces the three
acceptance routes with a single exact condition:
\begin{verbatim}
normalized(chain) == canonical.tag
and normalized(witness) == canonical.witness
\end{verbatim}
This stricter metric is closer to target imitation and penalizes valid but
non-canonical blocked derivations. The main metric is closer to tolerant
semantic checking, but does not prove minimality or uniqueness.

\subsection{Metric Interpretation}

Joint accuracy is defined as correct label plus faithful evidence. This gives answer-only joint accuracy 0 by construction, because answer-only systems do not emit evidence to verify. We therefore read joint accuracy together with raw accuracy throughout the paper: answer-only measures label prediction alone, while proof-only and \model{} are asked to produce checkable reasoning. Unknown faithfulness is the fraction of gold-\texttt{Unknown} examples for which the model predicts \texttt{Unknown} and emits an accepted abstention witness. Strict unknown faithfulness is the fraction of gold-\texttt{Unknown} examples for which the predicted tag and witness exactly match the canonical target.
