\section{Method}
\label{sec:method}

\subsection{Task and Output Format}

Each ProofWriter question contains an indexed theory \(T\) and a query literal
\(q\). Under open-world semantics, \texttt{True} means \(q\) is derivable,
\texttt{False} means the opposite literal is derivable, and \texttt{Unknown}
means neither is derivable. Negative literals are explicit facts or rule
conclusions; absence of a positive proof is not treated as negation.

\model{} uses one four-line format:
\begin{verbatim}
MODE: <PROVE|REFUTE|ABSTAIN>
LABEL: <True|False|Unknown>
CHAIN: <triple/rule chain or FAIL[...]>
WITNESS: <statement or missing support>
\end{verbatim}
\prove{} chains derive the queried literal, \refute{} chains derive the
opposite literal, and \abstain{} outputs name a failed rule or state that no
rule or fact supports the query.

\subsection{Constructing Missing-Support Targets}

Let \(C(T)\) be the least forward-chaining closure of the indexed facts and
rules. For true and false queries, we reuse benchmark proof traces. For unknown
queries, we search for a blocked derivation of \(q\). A \texttt{FAIL[ruleX]}
target means that ruleX can derive the current target under some grounding, but
one or more grounded antecedents are absent from \(C(T)\). A
\texttt{FAIL[no\_rule]} target means that no fact and no rule consequent can
support the current target. If several blocked derivations exist, we choose a
canonical one by minimizing the number of missing leaves, then failure depth,
then rule identifier. This gives a deterministic training target without
claiming that the explanation is unique or globally minimal.

The support-deletion stress test uses the same closure computation. We remove a
fact or rule from a gold proof chain and keep the mutant only when the original
literal and its opposite are both underivable. The retained examples are
therefore unsupported open-world cases.

\subsection{Verification}

For \prove{} and \refute{}, the verifier replays the predicted chain and checks
that it derives the claimed literal. For \abstain{}, the main verifier accepts a
prediction if it matches the canonical failure tag, matches the canonical
witness, or mentions all canonical missing leaves. This tolerant check allows
surface variation after lowercasing and whitespace normalization, but it does
not align paraphrased entities, infer rule-reference aliases, or establish
minimal causality. Section~\ref{sec:experiments} therefore reports a stricter
audit that requires exact tag and witness match. The verifier does not repair
outputs or search for alternative evidence at test time.
